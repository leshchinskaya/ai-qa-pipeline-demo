# QA Device Monitor — демо-стенд (реальное приложение на моках)

Это **то же самое приложение** `qa-device-monitor` с его настоящим дизайном и UX,
но с вырезанным Firebase и авторизацией. Никакого бэкенда — данные живут в памяти,
пользователь «входит» автоматически.

## Как это сделано (минимум изменений в коде)
Весь оригинальный код (`index.html`, `css/`, `js/`, вёрстка, стили, модули) скопирован **как есть**.
Заменена только граница интеграции:

- **Убраны** 3 CDN-скрипта Firebase (`firebase-app/auth/firestore-compat`).
- **Добавлен** `js/core/mock-firebase.js` — лёгкий мок SDK:
  - `firebase.auth()` — авто-вход фейкового пользователя `demo@test.dev` (экран логина пропускается);
  - `firebase.firestore()` — реактивная in-memory база (`collection/doc/where/orderBy/limit/onSnapshot/get/add/update/set/runTransaction/batch`, `FieldValue.serverTimestamp`, `Timestamp.fromDate`).
  - `onSnapshot` обновляется при записи → каталог сам перерисовывается после бронирования.
- Засеяно 5 устройств, 2 пользователя и пустые служебные коллекции.

Остальные экраны приложения (админка, чат, статистика и т.п.) остаются, но не нужны для демо.

## Запуск
Камере браузера нужен `https`/`localhost`, поэтому открывайте через локальный сервер:

```bash
cd qa-device-monitor-demo
python3 -m http.server 8080
# открыть http://localhost:8080
```

(Можно открыть и как файл — каталог/поиск/бронирование работают; камера сканера доступна только по localhost/https.)

## Сценарий демонстрации
1. **Каталог + поиск** — приложение сразу открывается на вкладке «Устройства» (без логина).
   Поиск в шапке (`Android 14`, `pixel`, `iPhone`…) фильтрует на лету — настоящий UX приложения.
2. **QR-бронирование** — кнопка «Сканировать QR»:
   - навести камеру на QR с ID устройства, **или**
   - ввести ID вручную в поле сканера (`DEV-001`) и нажать «Найти».
3. Откроется реальная шторка брони → «Использую в офисе» / «Забрать домой» (с датой) → «Подтвердить».
   Устройство в каталоге тут же становится забронированным на вас.
4. **Возврат** — отсканируйте/введите `DEV-004` (уже на вас) → «Вернуть».
5. **Занято / неисправно** — `DEV-003` занято Анной (только инфо), `DEV-005` неисправно (бронь недоступна).

## Устройства (моки, in-memory)
| ID | Устройство | Состояние |
|----|-----------|-----------|
| DEV-001 | iPhone 13 | свободно |
| DEV-002 | Samsung Galaxy S21 FE | свободно |
| DEV-003 | Google Pixel 6a | занято (Анна Петрова) |
| DEV-004 | Xiaomi Redmi Note 10 Pro | забронировано на вас |
| DEV-005 | Samsung Galaxy A50 | неисправно |

Чтобы изменить данные — правьте блок `SEED DATA` в `js/core/mock-firebase.js`.
Перезагрузка страницы сбрасывает все брони к исходным.

## Проверено автоматически (Playwright, через реальный UI)
- авто-вход, экран логина пропущен, каталог из реального приложения отрисован;
- поиск фильтрует устройства;
- полный QR-флоу: открыть сканер → ввести ID → найти → подтвердить → устройство забронировано,
  в историю записана бронь с `source: 'qr'`, каталог обновился вживую;
- неисправное устройство (`DEV-005`) — без формы брони;
- ноль ошибок JS и сетевых 404.
