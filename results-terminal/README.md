# results-terminal — логи прогона QA-пайплайна

Это **транскрипты терминала** для каждого шага конвейера автогенерации тестов
(команды `make stepN` и их вывод). Пример того, как из требований получаются
ручные тесты, автотесты, план доработок и матрица трассируемости.

Артефакты, на которые ссылаются логи, лежат в папке [`../results/`](../results/).
Инструменты (фреймворки `qa-autogenerator-manual-tests` и `e2e-test-framework`)
в публичный набор не входят — здесь только их **вывод**.

## Порядок шагов

| Шаг | Файл | Что делает |
|-----|------|------------|
| 1 | [step1.md](step1.md) | Ревью требований (ТЗ v1.1) → оценка 42% |
| 2 | [step2.md](step2.md) | Генерация ручных тест-кейсов (component + scenario) |
| 3 | [step3.md](step3.md) | Генерация e2e-автотестов из сценарных тестов |
| 3c | [step3-component.md](step3-component.md) | Генерация e2e-автотестов из компонентных тестов |
| 7 | [step7.md](step7.md) | План дореализации кода по отчёту тестов |
| 8 | [step8.md](step8.md) | Применение плана дореализации |
| 9 | [step9.md](step9.md) | Трассировка требований → код/тесты (матрица) |
| 10 | [step10.md](step10.md) | План улучшения покрытия |
| 11 | [step11.md](step11.md) | Применение плана улучшения покрытия |

> Шаги 4–6 (прогон автотестов и регресс) в этих логах не сохранены.
> Вывод приведён «как есть» (обезличенный) — пути, имена и реальная почта убраны.
